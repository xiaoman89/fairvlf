# FairVLF

**Fair Vision-Language Detection of AI-Generated Faces across the Generator Frontier**

This repository contains the official implementation of FairVLF, a vision-language
approach for fair AI-generated face detection. FairVLF addresses two distinct sources
of unfairness in VLM-based detectors: visual-side demographic shortcuts and
language-side prompt sensitivity.

> **Status:** Research code, actively under development. Numbers and pretrained
> checkpoints will be released upon publication. This README will be expanded as the
> project matures.

---

## Overview

Modern vision-language face forensics improves detection, explanation, and
attribution, but it remains unclear whether these VLM-based detectors are fair across
demographic groups, especially on modern full-face AI generators. FairVLF studies and
mitigates this gap.

**Key insight.** A traditional CNN/CLIP detector has one bias channel (visual). A VLM
detector has two: the same face asked *"Is this face real or fake?"* versus *"Is this
Black female face real or fake?"* can yield a different fake probability. FairVLF
constrains the detector so its real/fake decision rests on forensic evidence rather
than demographic appearance or demographic wording.

**Method.** FairVLF fine-tunes a vision-language backbone with LoRA under a combined
objective:

```
L = L_det + lambda_DPC * L_DPC + lambda_DSA * L_DSA
```

- `L_det` — detection loss (binary cross-entropy on a calibrated fake score).
- `L_DPC` — Demographic Prompt Consistency: a margin-tolerant penalty that suppresses
  language-side prompt sensitivity.
- `L_DSA` — Distributional Score Alignment: an MMD-based alignment of score
  distributions across demographic groups (visual-side fairness).

See [`docs/METHOD.md`](docs/METHOD.md) for the full method description.

## Data

FairVLF is trained and evaluated on **FairFrontier**, a controlled-demographic,
full-face, text-to-image fairness benchmark. Real faces are drawn from
[FairFace](https://github.com/joojs/fairface); fake faces are generated by six
text-to-image generators (SD1.5, SDXL, FLUX-schnell, FLUX-dev, SD3.5, Qwen-Image).

**The dataset itself is not included in this repository** (size and licensing). Place
your local copy under a path you configure in the YAML files; the data loader reads
from that path. See [`docs/DATA.md`](docs/DATA.md) for the expected directory layout
and the manifest format.

## Quick start

```bash
# 1. Install
pip install -r requirements.txt

# 2. Verify the pipeline on a tiny subset (no real training, just a smoke test)
python -m fairvlf.train --config configs/smoke_test.yaml

# 3. Full training (after the smoke test passes)
python -m fairvlf.train --config configs/default.yaml
```

All runtime behaviour is controlled by the YAML config — no editing of source code is
needed to change hyperparameters, data paths, or the backbone.

---

## Reproducibility

Every run writes a timestamped directory under `runs/` containing the resolved config,
a full log, per-step metrics, and the git commit hash of the code that produced it.
Seeds are fixed and recorded. See [`docs/METHOD.md`](docs/METHOD.md) for details.
